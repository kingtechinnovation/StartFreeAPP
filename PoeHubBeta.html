Deploy a Poe Canvas App

Step-by-step guide to deploying your apps on PoeHub

Step 1 — Create an App Configuration

Go to the Apps tab and click "+ New App". Fill in the required fields:

App Name

A clear, descriptive name users will see (e.g. "AI Code Helper")

Bot Handle

The Poe bot this app talks to (e.g. Claude-Sonnet-4.5, GPT-5.2, Imagen-4-Ultra)

Category

Group apps by purpose: Productivity, Creative, Development, Research, etc.

System Prompt

Optional instruction that shapes the bot's behavior for this specific app

Parameters (JSON)

Bot-specific settings like {"thinking_budget": 8192} or {"aspect_ratio": "16:9"}

Step 2 — Publish Your App

After creating an app, it starts in Draft mode. Click the "Publish" button on the app card to make it Live — users will then see it in their App Catalog.

Tip: You can unpublish anytime to hide an app from users while you make changes. Users currently in a chat session won't be interrupted.

Step 3 — Users Launch & Use Apps

When a user clicks on a published app in their catalog, a chat interface opens. Their messages are sent to the configured Poe bot via the Poe Embed API (window.Poe.sendUserMessage), and streamed responses appear in real-time.

Step 4 — User API Key (Optional)

Users can enter their own Poe API key in Settings → Poe API Key. This key is stored only in memory for the current session and is never persisted or sent externally. It can be used for additional API authentication flows if you extend this portal.
